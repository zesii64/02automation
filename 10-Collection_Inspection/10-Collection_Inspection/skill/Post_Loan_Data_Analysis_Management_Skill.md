# 贷后数据分析管理 (Post-Loan Data Analysis & Management Skill)

> **目标**：在贷后巡检与日常看数中，统一使用「风险口径 + 回收口径 + 过程口径」三条线及完整归因逻辑，支持日报/周报与核心数字资产沉淀。

## 1. 三条线（与取数底表一致）

| 口径     | 底表                  | 汇报用语           |
|----------|-----------------------|--------------------|
| 风险口径 | vintage_risk          | 风险/资产质量与客群 |
| 回收口径 | natural_month_repay   | 回收结果           |
| 过程口径 | process_data          | 策略执行/过程指标   |

## 2. 业务归因逻辑（看数做分析 checklist）

- **风险口径**：整体风险 → 新老客 → due date/week/month 趋势 → 同比环比 → 金额段/期数/期限 → 客群画像变化（如 MOB1 好客户占比）→ 分层是否倒挂。
- **回收口径**：序时回收进度 → 自然月回收 → 哪段/哪批差、哪段有机会。
- **过程口径**：跟进次数、覆盖率等 → S0/S1/S2/M1 各阶段表现 → 各组表现与过程指标 → 组内金额段排序与机会点 → 渠道、发送量、接通率是否正常。

**完整清单**：见 [WIKI/01_Business_Design.md](../WIKI/01_Business_Design.md) 第二节「业务归因逻辑（完整）」。

## 3. 异常规则（以波动为主）

- 采用**相对变化**（如日/周环比），先以 10% 为示例阈值，后续按历史方差定正式阈值。
- 模型/分段倒挂：高分段逾期率不得高于低分段。

## 4. 维度与归因新增约定

- **后续新增维度、新增归因问题**：统一在 **WIKI/01_Business_Design.md** 的「业务归因逻辑」中新增，保持单一信息源。
- Skill、Wiki、报表与核心数字资产均以该文档为准；新增后同步更新本 Skill 的引用说明即可。

## 5. 产出形态

- **日报**：按三条线 + 归因逻辑产出当日巡检 HTML。
- **周报**：按同一逻辑做周维度聚合与趋势，与日报结构一致、颗粒度不同。
- **核心数字资产**：本业务设计可同步至 `11-Agent/Core_Digital_Assets` 作为贷后分析管理方法论资产。
